Sample Projects

How to Build Samples on Windows XP, Vista and Windows 7

Due to security/permissions of the projects (enfornced by Windows Security), it is recommended that the user copy the SDK's include, lib and sample folder into a fully writable user folder, like Documents or Desktop before attempting to build and/or run the samples.

The Delphi examples require building with Delphi 4 or higher, however, some later versions of Delphi require adding the unit Variants to the samples.

The C/C++ samples require building with Microsoft Visual Studio 6 or higher.

Samples

The WordPerfect Office X6 SDK provides the following sample projects for the SDK.

  • WordPerfect Third Party Handler (T.P.H.) in C++ -- shows how C++ can be used to create a third-party handler DLL for WordPerfect
    • Location: /Samples/TPH/CPPTPH/CPPTPH.dsw
    • To test, build the project and use regsrv32 to register the DLL. A menu item flyout (Multiple Clipboard) will be added to the WordPerfect Edit menu with subitems to insert/save selections to a file.

  • WordPerfect Third Party Handler (T.P.H.) in C -- shows how C can be used to create a third-party handler DLL for WordPerfect
    • Location: /Samples/TPH/CTPH/CTPH.dsw
    • To test, build the project and use regsrv32 to register the DLL. A menu item flyout (Multiple Clipboard) will be added to the WordPerfect Edit menu with subitems to insert/save selections to a file.

  • WordPerfect Third Party Handler (T.P.H.) in Delphi -- shows how Delphi can be used to create a third-party handler DLL for WordPerfect using the SDK
    • Location: /Samples/TPH/DelphiTPH/DelphiTPH.dpr
    • To test, build the project and use regsrv32 to register the DLL. This T.P.H. DLL demonstrates overriding existing commands such as open, save and save as in WordPerfect.

  • WordPerfect Office OLE automation example in Delphi -- shows how Delphi can be used to access a database (Customer.db) to read data, create WordPerfect documents and merge data files, and to create a Quattro Pro spreadsheet from the database data.
    • Source Location: /Samples/OLE/Delphi/DelphiOLE.dpr
    • Runtime requires: Requires Borland Database Engine (BDE) to be installed.
    • To test:
      1. Run the SetSDKAlias.exe in the copied version of Samples\DB folder.
        Open a command line window (CMD), and change to the folder where the SetSDKAlias.exe is located. Then type in the following line:

        setSDKAlias.exe %CD%

      2. Build the sample with Delphi (DelphiOLE.dpr)
      3. Copy the welcome.wpt and wpformat.dll from the project folder and Samples\Lib folder respectively to the same folder location as DelphiOLE.exe.
      4. Run DelphiOle.exe to test.
