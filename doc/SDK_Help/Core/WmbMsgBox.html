Using PerfectFit : Creating message boxes : WmbMsgBox function

WmbMsgBox function

The WmbMsgBox function displays a PerfectFit message box. It also allows you to pass argument lists for the messagebox to display. These arguments are displayed in the order specified in the parameter lpazMsg.

Prototype
WORD FAR PASCAL WmbMsgBox (  
HANDLE hCaller,  
HANDLE hLangResInst,  
LPAZ lpazCaption,  
LPAZ lpazMsg,  
DWORD dwFlags,  
LPAZ FAR* lprglpArgList,  
HWND hwndParent );
Parameters

hCaller

Specifies the handle of the instance invoking the message box

hLangResInst

Specifies the handle of the instance where the string is located

lpazCaption

Specifies the far pointer to the caption string

lpazMsg

Can be either an index to a string table or a far pointer to a message string.

 

If the high word of lpazMsg is 0, then this value is an index to a string table, and WmbMsgBox loads the string from the table. If the message should contain filenames or other replaceable parameters, use ^0, ^1, and so on to indicate the placement of these parameters (specified with lprglpArgList). When the message is displayed, ^0 is replaced by the first parameter of lprglpArgList, ^1 is replaced by the second parameter, and so on.

A replaceable parameter can appear any number of times in the message string, such as the following: "File ^0 was not found in the current folder (^1). Search for ^0 in other directories?"

All previous replaceable parameters must be specified. For example, you cannot specify ^4 unless ^0, ^1, ^2, and ^3 have been defined in lprglpArgList.

 

Optionally, this value is a far pointer to an array of far pointers to replaceable parameters for message.

hwndParent

Specifies the window handle of the parent

dwFlags

Specifies message-box flags. All Windows MessageBox flags are supported. In addition, you can specify the following PerfectFit flags:

  • WPMB_BUTTONMASK
  • WPMB_ICONMASK
  • WPMB_DEFBUTTONMASK
  • WPMB_MODALITYMASK
  • WPMB_MISCMASK
  • WPMB_ANSISTRING
  • WPMB_NOSAFETY
  • WPMB_BEEPONERROR
  • WPMB_DONTRETRY
  • WPMB_RECORDABLE
  • WPMB_NULLPARENT

The Windows flags are as follows.

Value
Meaning

MB_ABORTRETRYIGNORE

Specifies that the message box contains three buttons: Abort, Retry, and Ignore

MB_APPLMODAL

Specifies that the user must respond to the message box before continuing work in the window identified by the hwndParent parameter. However, the user can move to the windows of other applications and work in those windows. MB_APPLMODAL is the default if neither MB_SYSTEMMODAL nor MB_TASKMODAL is specified.

MB_DEFBUTTON1

Specifies that the first button is the default. Note that the first button is always the default unless MB_DEFBUTTON2 or MB_DEFBUTTON3 is specified.

MB_DEFBUTTON2

Specifies that the second button is the default

MB_DEFBUTTON3

Specifies that the third button is the default

MB_ICONASTERISK

Is the same as MB_ICONINFORMATION

MB_ICONEXCLAMATION

Specifies that an exclamation-point icon appears in the message box

MB_ICONHAND

Is the same as MB_ICONSTOP

MB_ICONINFORMATION

Specifies that an icon consisting of a lowercase letter `i' in a circle appears in the message box

MB_ICONQUESTION

Specifies that a question-mark icon appears in the message box

MB_ICONSTOP

Specifies that a stop-sign icon appears in the message box

MB_OK

Specifies that the message box contains one button: OK

MB_OKCANCEL

Specifies that the message box contains two buttons: OK and Cancel

MB_RETRYCANCEL

Specifies that the message box contains two buttons: Retry and Cancel

MB_SYSTEMMODAL

Specifies that all applications are suspended until the user responds to the message box. If the application specifies MB_ICONHAND, the message box does not become modal until after it is created; consequently, the parent window and other windows continue to receive messages resulting from its activation. System-modal message boxes are used to notify the user of serious, potentially damaging errors that require immediate attention (for example, running out of memory).

MB_TASKMODAL

Is the same as MB_APPLMODAL, except that all the top-level windows belonging to the current task are disabled if the hwndParent parameter is NULL. This flag should be used when the calling application or library does not have a window handle available but still needs to prevent input to other windows in the current application without suspending other applications.

MB_YESNO

Specifies that the message box contains two buttons: Yes and No

MB_YESNOCANCEL

Specifies that the message box contains three buttons: Yes, No, and Cancel

Return values

The return value is zero if there is not enough memory to create the message box. Otherwise, it is one of the following menu-item values returned by the message box.

Value
Meaning

IDABORT

The Abort button was selected

IDCANCEL

The Cancel button was selected. If the Cancel button is present, pressing the ESC key selects it.

IDIGNORE

The Ignore button was selected

IDNO

The No button was selected

IDOK

The OK button was selected

IDRETRY

The Retry button was selected

IDYES

The Yes button was selected
