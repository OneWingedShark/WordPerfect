Using Quattro Pro import functions

This section provides the following information for the qprowin.def import functions: name, prototype, description, parameters, return values, and relevant sample and lesson files. The sample files are provided with the Corel WordPerfect Office SDK.

Some of the import functions use defined structures. The format of these structures can be found in the qprowin.h file provided with the SDK for WordPerfect.

As an object-oriented product, Quattro Pro has a variety of objects and associated properties available for use in the qprowin.def import functions QProSetProperty, QProGetProperty, and QProExecute. The objects and properties available are listed in the Quattro Pro Help. Two functions that are included in Quattro Pro - @PROPERTY and @COMMAND - should be reviewed, as they can assist in getting the proper syntax for the property of the object you are reading or setting. Starting with version 5.0, changes have been made to allow QProSetProperty, QProGetProperty, and QProExecute to be fully compliant with the Quattro Pro @Property macro command. For more information, please see the documentation for @Property and @Command in the Quattro Pro Help.

Some functions use the NotebookIndex parameter. For version 5.0 and later, the NotebookIndex value can be 0, to indicate to Quattro Pro that it should use the current Notebook index. For versions earlier than 5.0, all indexes were required to be valid, non-zero index values.
