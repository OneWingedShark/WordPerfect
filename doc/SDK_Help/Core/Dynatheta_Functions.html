Using PerfectFit : Setting up dynathetas

Setting up dynathetas

To understand dynathetas, you must first understand how commands are executed in a tokenized WordPerfect application.

Understanding tokenized applications

In a tokenized application, menu selections, toolbar selections, and macro commands are used to execute commands. These commands have low-level groups of information associated with them to provide a high-level user interface.

A theta is defined as a structured group of information, referenced by a command ID, containing the interface information associated with that ID. Most commands executed in tokenized WordPerfect applications are thetas. Here is some of the information contained in the theta structure:

  • names and parameters for macro commands
  • the name of the theta
  • text for the command's menu
  • a bitmap for the command's toolbar
  • text for the command's toolbar
  • Help text for the command's long prompt

A dynatheta (dynamic theta) is defined in the Windows Registry. Dynathetas give applications and third-party developers the ability to define thetas that act like base function thetas from the application's resource DLL and can be defined at any time. Thus, dynathetas allow dynamic augmentation of the resource DLL.

A dynatheta is a structure. The members of this structure are defined as follows:

ID

A unique number assigned by the application developer to each program action or command

Theta Name

A string that identifies the theta. This name represents the command in a written application macro.

Long Prompt

Text that describes the command. Sometimes called a Help prompt. The long prompt can be displayed on the status bar to indicate a command's function when it is highlighted by the cursor.

Toolbar Text

Text that can be displayed on a button

Toolbar Graphic

A bitmap image that can be displayed on a button

Menu Text

Text to be displayed on the menu. The text can optionally contain an ampersand (&) to indicate the mnemonic for the menu item.

Help File

Text that names the path and filename of the Help file to be used when Help is invoked for this command. If no file is specified, the default Help file is used.

Help ID

A unique alphanumeric sequence that identifies the Help topic to be displayed if Help is invoked. This value should not be translated for international versions.

Macro Definition

The parameter format for a command. An ID often requires parameters to indicate how the command should be processed. For instance, the parameter for FontBold indicates whether boldface formatting should be turned on or off.

Currently, the PerfectFit implementation does not allow the definition of parameters with a dynatheta. Most of the dynatheta functions require the atomAppName parameter to specify which application the dynatheta is being created for. If you are attempting to install a dynatheta and the application for which the dynatheta is being created is not running, you must use the Windows API GlobalAddAtom( ) function. When you call GlobalAddAtom( ), use the appropriate atom names for the application you are installing the dynatheta for:

  • WordPerfect - for WordPerfect
  • Presentations - for Presentations

After the dynatheta has been created and added to the Windows Registry, you may need to delete the dynatheta and re-create it. If you want to delete it manually, the dynathetas are stored in the following Windows Registry key:

\HKEY_CURRENT_USER\Software\Corel\PerfectScript\14\Commands\ 
application

The application key corresponds to the atom names listed in the table above.

Understanding dynatheta Functions

You can use the following dynatheta functions:

  • WpidDynaThetaAdd function
  • WpidDynaThetaGetModificationDate function
  • WpidDynaThetaListGetTempID function
  • WpidDynaThetaRemove function
  • WpidThetaCreate function
  • WpidThetaDestroy function
