Using PerfectFit : Providing file-management capabilities

Providing file-management capabilities

This API subsystem is a group of functions that let you provide file-management capabilities in your application without having to write all the code. In most cases, error and special-case handling is provided by the functions. Also provided are common Filespec and FileSaveAs dialog boxes, which give you access to Directory Management functions and Viewer Document management systems through Open Document Management Architecture (ODMA).

The WfsFileOpenEx( ) API, which is described in this section, is preserved mainly for backward compatibility. Using the PerfectFit FileOpen COM Object is the preferred method for interfacing with the Corel File Open dialog box.
Understanding File Open input flags

The File Open input flags are used with WfsFileOpenEx( ).

Flag
Desired result

Understanding File I/O input flags

The File I/O input flags are used by WfsFileCopy( ), WfsFileMove( ), WfsFileDelete( ), WfsFileAttr( ), WfsCreateDir( ), and WfsRemoveDir( ).

Flag
Desired result

FS_NOPROMPT_ERROR

Does not prompt if an error occurs

FS_NOPROMPT_CREATE

Does not prompt before creating an item

FS_NOPROMPT_REPLACE

Does not prompt to replace files. Note that if both replace and replaceall are not set, PerfectFit prompts for replace, but not replaceall. If replace is set and replaceall is not set, PerfectFit prompts for replaceall. If both are set, PerfectFit does not prompt for either replace or replaceall.

FS_NOPROMPT_REPLACEALL

Does not prompt for replaceall files. Note that if both replace and replaceall are not set, PerfectFit prompts for replace, but not replaceall. If replace is set and replaceall is not set, PerfectFit prompts for replaceall. If both are set, PerfectFit does not prompt for either replace or replaceall.

FS_NOPROMPT_READONLY

Does not prompt on move/delete of a read-only file

FS_NOPROMPT_HIDDEN

Does not prompt on move/delete of a hidden file

FS_NOPROMPT_SYSTEM

Does not prompt on move/delete of a system file

FS_NOPROMPT_STATUS

Does not display status spinner

FS_NOPROMPT_REPLACENORMAL

Does not prompt on replacement of a normal file

FS_NOPROMPT_REPLACEREADONLY

Does not prompt on replacement of a read-only file

FS_NOPROMPT_REPLACEHIDDEN

Does not prompt on replacement of a hidden file

FS_NOPROMPT_REPLACESYSTEM

Does not prompt on replacement of a system file

FS_PROMPT_YESTOALL

Prompts to delete all specified files (if set)

FS_NOPROMPT_DESTDEL

Does not prompt to delete existing files on the removable target (if set)

FS_NOCOPY_IDENTICAL

Does not copy files that have the same date and time as the target file (if set)

Understanding File I/O return values

The File I/O return values are used by WfsFileCopy( ), WfsFileMove( ), WfsFileDelete( ), WfsFileAttr( ), WfsCreateDir( ), and WfsRemoveDir( ).

Return value
Description

FS_RETURN_OKERROR

Indicates that an error occurred after a successful operation. Note that the FS_RETURN_OKERROR and FS_RETURN_OKCANCEL flags are used mainly for template operations. Either value returned means the operation was successful on one or more files before the error or cancel occurred.

FS_RETURN_OKCANCEL

Indicates that Cancel was selected after a successful operation. Note that the FS_RETURN_OKERROR and FS_RETURN_OKCANCEL flags are used mainly for template operations. Either value returned means the operation was successful on one or more files before the error or cancel occurred.

FS_RETURN_CANCEL

Indicates that Cancel was selected, so no successful operation was performed

FS_RETURN_ERROR

Indicates that an error occurred, so no successful operation was performed

FS_RETURN_N

Is reserved for unhandled

FS_RETURN_OK

Indicates that the operation was successful

FS_RETURN_ALLREADONLY

Indicates that deleting all read-only files was selected

FS_RETURN_ALLHIDDEN

Indicates that deleting all hidden files was selected

FS_RETURN_ALLSYSTEM

Indicates that deleting all system files was selected

FS_RETURN_REPLACEONE

Indicates that Yes replace was selected

FS_RETURN_REPLACENORMAL

Indicates that Yes-to-all replace was selected

FS_RETURN_REPLACEREADONLY

Replaces all read-only files

FS_RETURN_REPLACEHIDDEN

Replaces all hidden files

FS_RETURN_REPLACESYSTEM

Replaces all system files
