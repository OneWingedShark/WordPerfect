Using PerfectFit : Providing document conversion

Providing document conversion

WordPerfect has developed a series of conversion functions that ship as part of PerfectFit. These functions include conversions for popular word-processing formats such as Microsoft Word, Ami Pro, and Rich Text Format (RTF), as well as graphics, spreadsheet, and database formats.

Understanding the functions for providing document conversion

You can use the following functions for converting documents:

  • WcvtConvertFile function
  • WcvtConvertHandle function
  • WcvtDetectBufferFormat function
  • WcvtDetectFileFormat function

The document-conversion functions use the following flags when converting a file from one format to another. These flags can be combined to control processes such as file format, cursor appearance, and document prefix.

Flag
Description

CVT_NODEFAULTS

Does not write document defaults (such as default margins, default justification, and so forth) into the converted document. Conversion for the Clipboard always sets this bit so that paste operations work as expected.

CVT_NODISKGRAPHICS

Does not create external files for embedded graphics

CVT_PLEASEWAIT

Displays the Please Wait dialog box that allows the user to cancel the conversion

CVT_WINO

Specifies that the output file should contain the Windows native format only. This is used when converting a Windows metafile to WPG to indicate that the metafile should be encapsulated into the WPG but no conversion should be performed. Do not set both CVT_WINO and CVT_WINCV. If neither is set, the graphic is converted and the metafile is not encapsulated.

CVT_WINCV

Specifies that the output file should include the Windows native format along with the converted data. This is used when converting a Windows metafile to WPG to indicate that in addition to the converted graphic, the metafile should be encapsulated into the WPG. Do not set both CVT_WINO and CVT_WINCV. If neither is set, the graphic is converted and the metafile is not encapsulated.

CVT_SKIPPREFIX

Does not process the prefix. This also means that functions that refer to prefixes are handled differently.

CVT_LENGTH_IN

Specifies that the first four bytes of hMem contain the length of the data that follows the first four bytes (dwLen is ignored)

CVT_LENGTH_OUT

Places the length of the output data in the first four bytes of the output data. lpOutLen is this value plus four.
