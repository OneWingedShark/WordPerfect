Using PerfectFit : Providing file-management capabilities : wopen function

wopen function

The wopen function allows the user to open a file by using WordPerfect File I/O.

Prototype
WFP SHAPI wopen (  
WPFNAME name,  
WUDWORD flags );
Parameters

name

Specifies the path of the file to open or create passed in as a WPFNAME structure

flags

Passes the following flags, which determine the access of the file opened:

 
  • WIO_READ - Opens the file with read access

 
  • WIO_WRITE - Opens with write privileges

 
  • WIO_SHARED - Allocates memory for this file from shared memory

 
  • WIO_CREATE - Creates the file if it doesn't exist

 
  • WIO_EXCL - Opens only if the file doesn't exist previously

 
  • WIO_BUF - Buffers the file I/O

 
  • WIO_CACHE - Caches the file

 
  • WIO_DELETE - Automatically deletes the file when finished

 
  • WIO_TRUNC - Truncates the file to length 0 on open

 
  • WIO_MEMORY - Puts this file into memory, not onto disk

 
  • WIO_PERSISTENT - Makes this memory file stay around after closing

 
  • WIO_DENYNONE - File-sharing mode: Allows reading and writing

 
  • WIO_DENYREAD - File-sharing mode: Denies reading the file

 
  • WIO_DENYWRITE - File-sharing mode: Denies writing the file

 
  • WIO_DENYALL - File-sharing mode: Denies reading and writing of the file

 
  • WIO_EXPLODEPREFIX - Forces loading of the Prefix area

 
  • WIO_DOCSAVECHECK - Checks the file on saving to see if it is corrupt. Doesn't allow saving of corrupt documents.

 
  • WIO_DOCFILE - Creates the file as an OLE compound file. If file exists, it must already be a compound file. (This option can be used to create a WordPerfect 7 document.)

 
  • WIO_FLATTFILE - ALWAYS opens the file as a flat file

Return values

This function returns zero if an error occurs. If there was no error, it returns a WFP, which is a WordPerfect File Pointer to the opened file.
