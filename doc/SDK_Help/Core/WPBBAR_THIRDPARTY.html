Using PerfectFit : Creating toolbars : WPBBAR_THIRDPARTY structure

WPBBAR_THIRDPARTY structure

The WPBBAR_THIRDPARTY structure contains the handle to a toolbar window, a handle to the template (data store) where the toolbar is stored, the name of the toolbar, and the data store ID of the toolbar.

The information in this structure is passed to a third-party DLL as the first parameter of the token generated when a toolbar is being loaded (0xD40A) by a WordPerfect application (see the PerfectFit WOAPI sample, MenuTB.C). The token ID is defined in SCWIN.H as BUTTONBAR_LOAD. If you need to manipulate or change the toolbar after this token has passed through the third-party interface, you should save the information in this structure.

Structure
typedef struct{  
WORD dsidBBar;  
LPWZ lpwzName;  
HWND hwnd;  
HDS hds;} WPBBAR_THIRDPARTY, FAR *PWPBBAR_THIRDPARTY;
Members

dsidBBar

Data store ID of toolbar. Used internally by PerfectFit and WordPerfect applications.

lpwzName

Toolbar name (in WORD string format)

hwndBBar

Handle to the toolbar window. This window handle should be used when referencing a specific toolbar. Multiple toolbars can be displayed at any given time, so the Windows API FindWindow( ) function is not be a reliable method for getting the handle to the toolbar window.

hds

Handle to the template (data store) where the toolbar resource resides. This is used when saving the toolbar.
