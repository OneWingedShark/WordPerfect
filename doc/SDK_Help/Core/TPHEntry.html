Using entry points in Third Party Handler DLLs : Defining required functions in Third Party Handler DLLs : TPHEntry function

TPHEntry function

PerfectFit calls this function during DLL initialization. This function allows the DLL to perform any initialization.

This function is called before any PerfectScript commands are passed to the DLL. The function is provided with the application's resource language code to indicate which language is being used by the Corel WordPerfect Office application.

This function must be exported as ordinal value 4.

C Prototype
WORD FAR PASCAL TPHEntry ( WORD wLanguage );
Pascal Prototype
function TPHEntry ( wLanguage: WORD ): WORD;
Parameters

wLanguage

Specifies the default language of the calling application. For the English (EN) version, the value of the wLanguage parameter would be LOBYTE "E" HIBYTE "N".

Return values

A value greater than zero should be returned to indicate success. A return value less than or equal to zero indicates that an error occurred.

C sample
WORD FAR PASCAL TPHEntry (WORD wLanguage) 
{ 
  aMyAtom = GlobalAddAtom (szAppName); 
  /* Creates a dynatheta for WordPerfect. The theta will be used to */
  /* create a menu and button, which can then be monitored in the   */
  /* TPHHandleToken function in the DLL                             */  
  dWPWPThetaID = CreateTheta (szAppName, aWP, "About Third Party...", 
  "About TP", "Third Party About Dialog", "THIRD_PARTY", HELP_GRP, 
  TRUE); 
  return 1; 
}
Pascal sample
