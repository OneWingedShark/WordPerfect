Using PerfectFit : Creating custom controls : DDWDATA structure

DDWDATA structure

The DDWDATA structure is maintained for each drag/drop well control and is used to pass messages from the control to its parent and to receive data back from the parent.

Definition
typedef struct 
{ 
HWND hwnd; 
UINT uiId; 
DWORD dwRsvp; 
HINSTANCE hInstIcon; 
WORD idIcon; 
HICON hIcon; 
WORD wIndex; 
LPDDROPOBJECT lpObject; 
} DDWDATA, FAR* LPDDWDATA;
Types

DDWDATA

Specifies the DDWDATA structure

LPDDWDATA

Acts as a long pointer to a DDWDATA structure

Members

hwnd

Specifies the window handle of the drag/drop well control sending the message

uiId

Specifies the ID of the drag/drop well control sending the message

dwRsvp

On a WPMSG_DD_RSVP message, contains the response sent by the window that the cursor is over

hInstIcon

Specifies the instance handle for this icon. On the WPMSG_DD_ICONHINST message, the hInstance of the module containing the icon should be placed in this field.

The hInstance can be the actual hInstance or WPRES_TASK, WPRES_SHARED_RESOURCE, and so on.

idIcon

Specifies the identifier for icon (0 for a named icon)

hIcon

Specifies the handle to the icon that is displayed for this control

wIndex

Is used on WPMSG_DD_DEST messages from a drag/drop list box to indicate the item that the cursor is over

lpObject

Acts as a pointer to the object being dragged
