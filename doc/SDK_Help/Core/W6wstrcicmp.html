Using PerfectFit : Handling strings : W6wstrcicmp function

W6wstrcicmp function

The W6wstrcicmp function performs a case-insensitive WORD string comparison at most c CHARACTER items (not WORD items).

This function was written to solve a problem in the W6wstrnicmp function, which returns TRUE if the two strings are still equal after the specified number of WORD items in either string. In contrast, the W6wstrcicmp function returns TRUE if the two strings are still equal after the same number of CHARACTER items in both strings. As long as the strings being compared do not contain non-character codes that W6wstrFirstCh and W6wstrNextCh skip (such as [Tab] and [Bold]), these two methods return the same results on the same strings. If there are non-character codes in the strings, then W6wstrncmp ignores their values by skipping over them, but it counts them as part of the n WORD items to be compared.

Prototype
int FAR PASCAL W6wstrcicmp (  
LPWZ lpwzStr1,  
LPWZ lpwzStr2,  
int cc );
Parameters

lpwzStr1

Specifies a far pointer to WORD string 1

lpwzStr2

Specifies a far pointer to WORD string 2

cc

Specifies the count of characters (not WORD items) to compare

Return values

This function returns one of the following values:

  • -1, if string 1 < string 2
  • 0, if string 1 == string 2
  • 1, if string 1 > string 2
