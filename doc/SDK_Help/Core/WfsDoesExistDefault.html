WfsDoesExistDefault function

The WfsDoesExistDefault function validates the input file specification (path+filename, path, filename, template, and so forth) and determines whether it exists, using lpszDefault to fill in any missing components. It also parses the file specification into its path and filename/template components. The input file specification is assumed to be an ANSI string. The path can have a final backslash ( \ ). The lpszOutPath and lpszOutName parameters string can be passed in as NULL.

Prototype
short SHAPI WfsDoesExistDefault ( 
LPSTR lpszDefault, 
LPSTR lpszInSpec, 
LPSTR lpszOutPath, 
LPSTR lpszOutName );
Parameters

lpszDefault

Specifies the default value used to fill in missing components of lpszInSpec

lpszInSpec

Acts as a pointer to the file specification to validate

lpszOutPath

Acts as a pointer to a buffer that contains the drive/directory component

lpszOutName

Acts as a pointer to a buffer that contains the filename (or template) component

Return values

Returns one of the values listed in the following table.

Return value
Condition

FS_EMPTY

Indictates that the file specification was empty

FS_INVALIDDRIVE

Indictates that the specified drive was invalid

FS_INVALIDDIR

Indictates that the specified folder was invalid

FS_FILEDIRNOTFOUND

Indictates that the specified drive or folder was not found

FS_ISDIR

Indictates that file specification had a valid folder name (in lpszOutPath)

FS_ISFILENAME

Indictates that the file specification had a valid filename (lpszOutPath points to drive, full folder, and filename)

FS_ISTEMPLATE

Indictates that the file specification had (optionally) a valid drive/dir, and included a file template (for example, it had wildcard characters)

FS_INVALIDFILENAME

Indictates that the file specification had an invalid filename
