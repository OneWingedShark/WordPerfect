Using Dynamic Data Exchange (DDE) : Using Presentations as a DDE server

Using Presentations as a DDE server

Presentations can function as a DDE server.

Through PerfectScript, Presentations can act as a DDE client. For more information, see "Using PerfectScript to set up a DDE client."
Specifying the server name

Presentations supports the following server name for the DDE application: Presentations.

Using supported topics

Presentations supports the single topic command. This topic allows you to send any valid PerfectScript command string to Presentations for processing - for example, FileOpen( "myfile.wpg" ), AddSlide( 5 ), or EnvCurrentSlideNumber( ).

Sending commands

The client application can use its equivalent of the DDEExecute statement to send PerfectScript command strings to Presentations. Multiple PerfectScript commands can be concatenated and sent as a single DDEExecute transaction. An alternative method for executing commands is to use the equivalent of the DDERequest transaction, as described later in this section.

Requesting information

Presentations provides access to its system variables through DDE. A client application can use its equivalent of the DDERequest statement to send a system-variable query command to Presentations; Presentations then returns the value for the specified system variable as an ANSI string. Boolean values are returned as either "TRUE" or "FALSE", and numeric values are returned as their ANSI string representations (for example, "357" ). DDERequest transactions must be sent individually; they cannot be concatenated.

Using valid command strings

Presentations supports any syntactically correct PerfectScript command that adheres to the following criteria:

  • The command string must consist only of PerfectScript Presentations product commands.

VALID: FileOpenDlg()

VALID: AddSlide( 5 )

INVALID: ASSIGN( x; 5 ) AddSlide( x )

  • Presentation system variables (commands beginning with Env) are valid only with DDERequest transactions.

VALID: EnvCurrentSlideNumber

VALID: EnvPaths( Program! )

The command string cannot reference any PerfectScript variables.

VALID: AddSlide ( 5 )

INVALID: AddSlide ( x )

INVALID: FileOpen( vFile )

In addition to the Presentations product PerfectScript commands, Presentations also supports the MacroFilePlay command.

Using the MacroFilePlay( macroname ) command

When this command is sent, Presentations attempts to play the script specified in the macroname parameter. This parameter is a quoted string that should contain the full path and name of the PerfectScript macro. If you do not include the full path, Presentations attempts to locate the script by looking in the default macro folder and the Presentations program folder.
