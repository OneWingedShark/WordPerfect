Understanding Dynamic Data Exchange (DDE)

Dynamic Data Exchange (DDE) is a method of interprocess communication supported under Windows. It is a way in which Windows applications can communicate with each other. The applications communicate by establishing a DDE conversation and posting or sending messages to each other. In each DDE conversation, one application functions as the server while the other functions as the client. The server application has access to resources it can provide to the client. The client application requires some type of information-processing from the server.

To share information with Windows applications created before Windows 95, you must use DDE.
It is important to note that DDE is not the prefered method of interprocess communication. We recommend that you use OLE Automation. For information on OLE Automation, please see "Understanding OLE Automation."
