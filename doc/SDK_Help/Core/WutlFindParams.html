WutlFindParams function

The WutlFindParams function finds all the task-specific parameters. This is called by an application prior to link-window creation. It goes through all startup options as specified by the function's parameters. The application's parameter handler is called each time a parameter is found. The application-supplied parameter structure (hTaskParams) is used in the link-window structure to supply the knowledge of the application's task startup state to PerfectFit.

Prototype
UTLParamCntType SHAPI WutlFindParams (  
LPCSTR lpszCmdLine,  
LPDosEnvVars lpDosEnvVars,  
LPCSTR lpszEnvFile,  
UTLHTaskParams hTaskParams,  
UTLLPParamHandler lpParamHandler );
Parameters

lpszCmdLine

Specifies the command-line parameters (if any) passed upon application startup

lpDosEnvVars

Specifies any DOS-environment variables you want to pass to PerfectFit or to your program. If a parameter passed is application-specific, it is ignored by PerfectFit.

lpszEnvFile

Specifies the path and file name of the .env file used by your application.

hTaskParams

Specifies the handle to a memory block. This memory block holds the specific parameters for the task (that is, your program). These parameters include the .env file switches, DOS-environment variables, and the command-line parameters passed by your application.

lpParamHandler

Specifies an optional long pointer to a callback function that allows your application to process the task-specific parameters passed in

Return values

Number of parameters found is returned.
