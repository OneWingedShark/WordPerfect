Using PerfectFit : Creating toolbars : WPBBAR_BUTTON structure

WPBBAR_BUTTON structure

The WPBBAR_BUTTON structure contains the bitmap and text information needed by a program to define an individual button on the toolbar. For information how and when to use this structure, see the PerfectFit API functions WpidBBarGetBtnInfo( ) and WpidBBarSetBtnInfo( ).

Structure
typedef struct{   
WORD idButton;  
WORD fFlags;  
HBITMAP hbmpGlyph;  
char azButtonText[PID_MAXLENBBARTEXT+1];  
char cSizeIndex;  
RECT rect;  
WPBBAR_BTN_FACE face;  
HWND hwnd;  
DWORD dwReserved;  
DWORD dwUserData;  
DWORD fCtrlStyle;  
} WPBBAR_BUTTON, FAR *PWPBBAR_BUTTON,  
NEAR *NPWPBBAR_BUTTON, FAR *LPWPBBAR_BUTTON;
Members

idButton

Identifier for this button (token ID)

fFlags

Any combination of the flags in the table that follows

hbmpGlyph

Handle to the bitmap for this button

azButtonText[ ]

Text for this button

cSizeIndex

Index into WPBBAR_SIZEINFO

rect

Size and position of the button at the moment this message is sent. Coordinates are relative to the upper-left corner of the toolbar.

dwReserved

Reserved

dwUserData

Application defined

fCtrlStyle

Custom control styles

The button flags can have one or more of the following bits set:

Flag
Definition

BB_TOGGLEBUTTON

Specifies that the action of the button is such that it alternates between up and down whenever it is pressed

BB_BUTTONDISABLED

Specifies that the button is inactive and dimmed

BB_DORMANTSTATEDOWN

Specifies that a toggle button is in the down state

BB_ACTIVESTATEDOWN

Specifies the state of the button before the user has released the mouse
