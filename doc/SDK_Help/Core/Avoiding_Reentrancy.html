Avoiding DDE Reentrancy errors

During the processing of a DDE transaction (Request or Execute), Corel WordPerfect Office applications may yield processor time and allow other applications to run. This is significant in the event that the client sends DDE commands to Corel WordPerfect Office applications based on messages that the client receives in its window procedure. Although the Corel WordPerfect Office application is performing a command on the client's behalf, it is possible for the client to receive messages in its Window procedure before the call to DDE Execute or DDE Request has completed. Because DDE does not allow a client to send a command for a particular conversation while another command for the same conversation is in process, the client application should make sure that a DDE Execute or DDE Request command is not currently in process before sending another such command through DDE. Otherwise, a DDE Reentrancy error will result, and the command may not execute successfully.
