Using PerfectScript with OLE Automation

Understanding PerfectScript (see "Understanding PerfectScript macros") is the key to using OLE Automation to integrate with Corel WordPerfect Office.

The SDK provides two useful tools for working with PerfectScript:

  • the External Command Inserter
  • the Command Recorder
Understanding the External Command Inserter

The Command Inserter can be used to insert OLE Automation code for C++, Delphi, and Visual Basic either into the SDK application or directly into supported external developer applications. If you are familiar with PerfectScript, then using this feature will allow you to have more control over what commands are sent to the Corel WordPerfect Office application.

Currently, the External Command Inserter works with the following developer tools: Paradox, Quattro Pro, Microsoft Excel, Microsoft Visual Basic, and Borland Delphi. If you have a code window open in any of these applications (in Microsoft Excel and Quattro Pro, the spreadsheet is the code window), the External Command Inserter will insert the code into the window. If more than one of these applications are running, you are prompted to choose which tool you want to use to insert the code. If none of these applications are running when you try to use the External Command Inserter, an error message is displayed.

You can choose parameters for the commands you want to insert. Be sure to click Update after you select a parameter; otherwise, the parameter is not inserted into the PerfectScript string or, ultimately, the Code Window, and the resulting command may yield incorrect or unexpected results.

Occasionally, you may want to insert a PerfectScript command without its parameters. The External Command Inserter cannot insert blank parameters into the Code Window; however, it can insert a placeholder in the form of a semicolon ( ; ) so that PerfectScript can determine the order of the parameters it receives.

Understanding the Command Recorder

The Command Recorder allows you to output OLE Automation code that works with Corel WordPerfect Office, faster and easier than ever before. The Command Recorder can take actions performed in WordPerfect or Presentations and insert the equivalent OLE Automation code into the SDK application or external applications such as C++, Delphi, or Visual Basic.

If you perform any unnecessary steps during the recording process, you must edit the OLE Automation code generated by the Command Recorder.
The Command Recorder requires you to use the values for the enumerations sent through OLE Automation; you cannot use the named enumerations used by PerfectScript. For example, the FileOpen PerfectScript command has two parameters: the filename and an enumerated value that denotes the current format of the document. In PerfectScript, you can specify the named enumeration for the format of the document (for example, WordPerfect_6). However, for OLE Automation, you must use the associated enumeration value (for example, 3); otherwise, WordPerfect cannot execute the command. To insert enumeration values rather than named enumerations, use the External Command Inserter.
