Using PerfectFit : Providing document conversion : WcvtConvertHandle function

WcvtConvertHandle function

The WcvtConvertHandle function converts the data referenced by a memory handle from one format to a handle that references data in another format

Prototype
WORD FAR PASCAL WcvtConvertHandle ( 
HGLOBAL hMem, 
WORD wInFormat, 
DWORD dwLen, 
WORD wOutFormat, 
WORD wFlags, 
CVTINFO far* lpCvtInfo, 
LPHANDLE lpHOut, 
LPDWORD lpOutLen );
Parameters

hMem

Specifies the handle to the data to be converted (in file format)

wInFormat

Specifies the format of the input file, or CV_AUTODTKT (1020) to automatically detect the input format

dwLen

Specifies the length of the data to be converted (does not need to include the null on null-terminated data types). If you pass 0 for this parameter and CVT_LENGTH_IN is not set, this function attempts to calculate the proper length. For text formats (ASCII, ANSI, RTF) the actual string length is used. For other formats, GlobalLength on the hMem is used, which may be slightly larger than the actual data length.

wOutFormat

Specifies the desired output format

wFlags

Can be zero or any combination of the following flags:

 

CVT_NODEFAULTS

 

CVT_NODISKGRAPHICS

 

CVT_LENGTH_IN

 

CVT_LENGTH_OUT

 

CVT_PLEASEWAIT

 

CVT_PLEASEWAIT

 

CVT_WINO

 

CVT_WINCV

 

CVT_SKIPPREFIX

lpCvtInfo

Is always NULL

lpHOu

Specifies the pointer to the location in which to store the handle to the converted data

lpOutLen

Specifies the pointer to the location in which to store the length of the converted data (not including any possible null terminator)

Return values

If successful, this function returns zero along with a new global memory handle to the converted data (lpHOut) and the length of the data (lpOutLen).

The output data always contains at least one null byte or word at the end so that text formats are properly null-terminated. The length that is returned does not include this null.
