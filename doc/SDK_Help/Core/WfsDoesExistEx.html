WfsDoesExistEx function

The WfsDoesExistEx function validates the input file specification (path+filename, path, filename, template, and so forth) and determines whether it exists. The input file specification is assumed to be an ANSI string. The path can have a final backslash ( \ ). The lpszOutSpec parameter can be passed in as NULL.

Prototype
short SHAPI WfsDoesExistEx ( 
LPSTR lpszInSpec, 
LPSTR lpszOutSpec );
Parameters

lpszInSpec

Acts as a pointer to the file specification to validate

lpszOutSpec

Acts as a pointer to a buffer which that contains the drive/directory/filename components

Return values

Returns one of the following values.

FS_EMPTY

Indicates that the file specification was empty

FS_INVALIDDRIVE

Indicates that the specified drive was invalid

FS_INVALIDDIR

Indicates that the specified folder was invalid

FS_FILEDIRNOTFOUND

Indicates that the specified drive or folder was not found

FS_ISDIR

Indicates that file specification had a valid folder name (in lpszOutPath)

FS_ISFILENAME

Indicates that file specification had a valid filename (lpszOutPath points to drive, full folder, and filename)

FS_ISTEMPLATE

Indicates that file specification had (optionally) a valid drive/dir, and included a file template (for example, it had wildcard characters)

FS_INVALIDFILENAME

Indicates that file specification had an invalid filename
