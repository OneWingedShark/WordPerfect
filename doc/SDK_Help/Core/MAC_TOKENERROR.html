MAC_TOKENERROR functional element

The MAC_TOKENERROR functional element is an enumeration list used to define the various error conditions resulting from processing a PerfectScript command.

C enumerations
typedef enum 
{ 
MAC_FUNCTION_OK = 0, 
MAC_FUNCTION_UNKNOWN = -99, 
MAC_FUNCTION_NOTFOUND, 
MAC_FUNCTION_CANCEL, 
MAC_FUNCTION_ERROR, 
MAC_FUNCTION_INVALID_PARM, 
MAC_FUNCTION_INVALID, 
MAC_FUNCTION_NOT_HANDLED, 
MAC_FUNCTION_RETURN_LATER 
} MAC_TOKENERROR;
Members

MAC_NO_ERROR

Indicates that the function was successful

MAC_FUNCTION_UNKNOWN

Indicates that the command is unknown

MAC_BASE_GENERAL_ERRORS

Indicates that the ommand is unknown

MAC_FUNCTION_NOTFOUND

Indicates that the unction resulted in a Not Found condition

MAC_FUNCTION_CANCEL

Indicates that the unction resulted in a Cancel condition

MAC_FUNCTION_ERROR

Indicates that the function resulted in an Error condition

MAC_FUNCTION_INVALID_PARM

Indicates that the function contained an invalid parameter

MAC_FUNCTION_INVALID

Indicates that the function is invalid at this time

MAC_FUNCTION_NOT_HANDLED

Indicates that the function was not handled

MAC_FUNCTION_RETURN_LATER

Used by PerfectFit to delay returning from command-handling functions
