TPHVersion function

PerfectFit calls this function during DLL initialization. This function must be defined even though it is currently not used by PerfectFit.

TPHVersion allows a Third Party Handler DLL to register its major and minor version numbers with PerfectFit. After the version numbers are registered, a value greater than 0 is returned.

The ordinal value for TPHVersion must be 1. The ordinal value defines the location of the function name in the application's string table.

C Prototype
DWORD FAR PASCAL TPHVersion ( VOID );
Pascal Prototype
function TPHVersion: DWORD;
Parameters

None

Return values

Any positive value

C sample
DWORD FAR PASCAL TPHVersion ( VOID ) 
{ 
  return 1L; 
}
Pascal sample
function TPHVersion: DWORD; 
begin 
  result := 1; 
end;
