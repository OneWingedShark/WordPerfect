Using PerfectFit : Creating dialog boxes

Creating dialog boxes

The Dialog Box Manager (DBM) is a collection of application programming interface (API) functions that gives third-party developers the capability to produce the look and feel of WordPerfect application dialog boxes. The DBM provides a sculptured look for edit controls and list boxes, recessed and shadowed static controls, and pop-up menus attached to controls. This allows third-party developers to create the impression that their dialog boxes are an extension of a WordPerfect application.

For more information on the available controls, along with the styles and messages used by the controls, see "Creating custom controls."

The DBM functions can be used with three types of programs: a Windows EXE program, a standard Windows DLL, or a Windows Open Application Program Interface (WOAPI) DLL program.

If the program is an EXE or a standard DLL, it must register with PerfectFit to use the DBM functions. For information on registering with PerfectFit, see "Registering PerfectFit."

If the program is a WOAPI DLL attached to a WordPerfect product, it does not need to perform this registration step. For more information on WOAPI DLLs, see "Understanding Windows Open API (WOAPI)."

With any type of program, as with any other Windows program with dialog boxes, functions for your DBM dialog boxes must be listed in the EXPORT section of your .DEF file.

Using DBM functions to create dialog boxes

This group of DBM functions covers the creation, manipulation, and destruction of the actual PerfectFit dialog boxes.

Several of these functions have working examples in the sample program that is provided with this SDK. If a function specifies that it has working examples, you can find these example functions in the sample program by searching for the function names: load the source code of the sample program into a text editor, and use the text-editor search capabilities to search for the function name.

You can use the following DMB functions for creating dialog boxes:

  • DBMCenterWindow function
  • DBMCreateDialogParam function
  • DBMCreateDialogParam2 function
  • DBMDefDlgProc function
  • DBMDialogBoxParam function
  • DBMDialogBoxParam2 function
  • DBMEndDialog function
  • DBMForceDlgRepaint function
  • DBMGetDlgBGColor function
  • DBMGrayString function
  • DBMIsDialogMessage function
  • DBMIsDialogModal function
  • DBMIsSculpturingSupported function
  • DBMPositionWindow function
  • DBMQueryDlgID function
  • DBMQueryDlgULong function
  • DBMQueryStandardDialogs function
  • DBMSetDlgULong function
  • DBMStandardDialog function
  • DBMUseStandardDialogs function
Using DBM functions to add control borders to dialog boxes

You can use borders to create three-dimensional frames around the controls in your DBM dialog boxes.

You can use the following DBM functions for adding control borders to your dialog boxes:

  • DBMDraw3DBorder function
  • DBMGet3DBorderSize function
  • DBMQueryCtl3DBorderFlag function
  • DBMSetCtl3DBorderFlag function
Using DBM functions to add pop-up menu button controls to dialog boxes

A pop-up menu button control is actually two controls: a button control and a menu control. DBM combines them for you to create a pop-up menu button control. It is similar to a combo box or (drop-down) list box control; however, instead of a list of data items, a pop-up menu button control has a list of menu items.

You can use the following DBM functions for adding pop-up menu button controls to your dialog boxes:

  • DBMCheckPopupMenuItem function
  • DBMDeletePopupMenuItem function
  • DBMDrawPopupBitmap function
  • DBMForcePopupButtonRepaint function
  • DBMGetCheckedMenuInfo function
  • DBMGetCheckMarkInfo function
  • DBMGetPopupBDownCBack function
  • DBMGetPopupButton function
  • DBMGetPopupDWORD function
  • DBMGetPopupHIWORD function
  • DBMGetPopupLOWORD function
  • DBMGetPopupMenu function
  • DBMInsertPopupMenuItem function
  • DBMIsPopupOwnerDraw function
  • DBMPopupMenuItemExists function
  • DBMQueryButtonOverhead function
  • DBMSetPopupBDownCBack function
  • DBMSetPopupDWORD function
  • DBMSetPopupHIWORD function
  • DBMSetPopupLOWORD function
  • DBMSetPopupMenu function
Using DBM functions to add recessed box controls to dialog boxes

A recessed box control forms a box whose surface appears to be indented from the surrounding surface of the dialog box.

You can use the following DBM functions for adding recessed boxes to your dialog boxes:

  • DBMGetRecessedBoxBrush function
  • DBMGetRecessedBoxClientRect function
  • DBMGetRecessedBoxTextColor function
Using DBM functions to add status bar controls to dialog boxes

A status bar is the control you see at the bottom of the WordPerfect editing window. DBM allows your program to create a similar status bar for use in your custom dialog boxes.

You can use the following DBM functions for adding status bar controls to your dialog boxes:

  • DBMGetSBarDC function
  • DBMGetSBarHeight function
  • DBMGetSBarRect function
  • DBMReleaseSBarDC function
  • DBMSetSBarPaintProc function
Using DBM functions to add shadow box controls to dialog boxes

This group of functions governs shadow box controls. Shadow box controls look like the box is "floating" over the surface of the surrounding dialog box.

You can use the following functions for shadow box controls:

  • DBMGetShadowboxBrush function
  • DBMGetShadowboxClientRect function
  • DBMGetShadowboxTextColor function
Understanding DBM messages

You can send the following messages to DBM-style controls:

  • DBM_POPM_BUTTON_GLYPH message
  • DBM_POPM_ENABLE message
  • DBM_POPM_SETBBITMAP message
  • DBM_POPM_SETBSTRING message
  • DBM_POPM_SETFOCUS message
  • DBM_POPM_SHOW_GLYPH_ONLY message

For each message, the wParam and lParam values are documented.
