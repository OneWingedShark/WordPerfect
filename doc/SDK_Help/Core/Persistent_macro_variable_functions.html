Using PerfectFit : Using persistent macro variables

Using persistent macro variables

There are three macro variable pools used by PerfectFit: local, global, and persistent. Local variables are accessible only when a macro is being executed (not by using the RUN or CHAIN commands). Global variables are available while a macro is being executed or when one macro runs or nests to another macro (by using the RUN or CHAIN macro commands). Persistent variables are accessible during the execution instance of the Macro Facility.

This section addresses persistent variables.

Local and global variables are accessible only by the Macro Interpreter and are available to third-party developers only through the macro language.
