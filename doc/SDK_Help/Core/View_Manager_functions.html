Using PerfectFit : Providing access to viewer functionality through the View Manager

Providing access to viewer functionality through the View Manager

This section provides API specifications for the interface between an application and the View Manager.

Applications interact with a module referred to as the View Manager. The View Manager processes requests from applications. Usually, this involves routing a View File request to an appropriate Viewer DLL, which provides the actual file viewing capabilities. WordPerfect products ship with several Viewer DLLs. The Convert subsystem of PerfectFit is used to provide viewing capabilities for many other file types by temporarily converting the file to a viewable format.

When a user clicks a view button, WordPerfect uses function calls to PerfectFit to tell the View Manager that it wants to view a file. These same function calls have been exposed so that third-party developers can use PerfectFit to display the contents of supported file formats.

A typical interaction between an application and the View Manager involves registering with PerfectFit, creating a View Manager and Viewer window, passing the name of the file to the View Manager, and destroying the Viewer window and the View Manager.

Registering with PerfectFit

An application must register with PerfectFit before any of the Viewer routines are called. The PerfectFit initialization routine registers the window types required by the View Manager. For information about registering an application, see "Registering PerfectFit."

Registering with PerfectFit is not required for Windows Open API DLLs. For information about Open API DLLs, see "Understanding Windows Open API (WOAPI)."

Creating a View Manager and Viewer window

An application typically requests creation of a View Manager when the user clicks a button on an application screen (usually a View button). Parameters to this request include the location and size for the Viewer window and any special window style options desired.

Passing the name of the file to the View Manager

Applications typically use the View Manager to view a specific file. Typically, the application dialog box contain a File List control, and the application sends a View File request to the View Manager each time the user highlights a file in the File List.

Destroying the Viewer window and the View Manager

The user typically destroys the Viewer window by double-clicking the system menu icon of the Viewer window (or by single-clicking it and selecting Close). However, if necessary, an application can send a Destroy request to the View Manager when desired. Commanding the view to be destroyed is recommended, but optional.

Understanding View Manager functions

This section defines the API functions that an application uses to interact with the View Manager.

Applications can use the following API functions to interact with the View Manager:

  • ViewMgrCreate function
  • WvwrViewFile function
  • WvwrViewFileEnd function
  • WvwrViewMgrDestroy function
