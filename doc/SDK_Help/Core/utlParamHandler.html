utlParamHandler function

The utlParamHandler function acts as a callback that lets your application process the task parameters passed to the PerfectFit link window. These parameters are processed after PerfectFit has processed them.

Prototype
BOOL FAR PASCAL utlParamHandler (  
LPCSTR lpszTag,  
LPCSTR lpszValue,  
UTLParamPriority uPriority,  
UTLParamSeq uSequenceNumber );
Parameters

lpszTag

Specifies the tag, or label, of a parameter. For example, in the parameter /nt-1, nt is the tag. The / and - (or =) are not passed in any parameters. If there is no tag (for example, c:\filename), an empty string is returned.

lpszValue

This is the value being passed by a parameter. In the example /nt-1, 1 is the value. If there is no value (for example, /f), this variable points to an empty string.

uPriority

Some WPGroup parameters must be executed in a particular order, no matter in what order they appear on the command line. This parameter returns the assigned priority of a WPGroup parameter. This is not useful to third-party developers.

uSequenceNumber

This is the original order of the parameter. This order is determined by the order it was found and by where it was found (for example, in the command line or in the .env file). This information is not useful to third-party developers.

Return values

None
