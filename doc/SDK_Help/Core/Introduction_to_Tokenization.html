Understanding tokenization

Many Corel WordPerfect Office application APIs are based on the concept of tokenization. A tokenized application is one where low-level events, such as a mouse click or key press, are packaged into a meaningful representations or tokens, such as "Save the current file" or "Send a new mail message." These tokens are then processed by the application.

Tokenization offers many advantages to an application. Most importantly, tokenization allows for a robust integration environment to be created.

Understanding the terminology of tokenization

In the Corel WordPerfect Office SDK documentation, you will notice that the terms "token," "command," and "macro command" are used interchangeably. These terms are equivalent and refer to high-level application events.

You will also find reference to a data structure called a "theta" or a "dynatheta." These terms refer to the specific data structure used to represent a token or command inside of the Corel WordPerfect Office applications.

Understanding the practical uses for a tokenized interface

Using the development environment for Corel WordPerfect Office applications requires frequently sending tokens to Corel WordPerfect Office applications. This can be done in many ways, depending on the circumstances involved;

  • by playing a PerfectScript macro
  • by sending Dynamic Data Exchange (DDE) commands
  • by using the Windows Open API
  • by using the Command Interface

Developers are free to mix and match these methods as necessary. However, not all methods are supported by all Corel WordPerfect Office applications.
