Understanding bitmap display controls

A bitmap display control is used to display a bitmap - transparently on the background, if desired. A bitmap display control is used like a static control; it displays the bitmap indicated in its client area.

The class name for bitmap display controls is WCWP_BITMAP.

The following styles are available to bitmap display controls.

Style
Effect

BMPS_SHARED_ RESOURCE

The bitmap specified in the lpszName of the CreateWindow( ) call references a bitmap in the WPRES_SHARED_RESOURCE resource. This is the default.

BMPS_TASK_ RESOURCE

The bitmap specified in the lpszName of the CreateWindow( ) call references a bitmap in the WPRES_TASK_RESOURCE resource.

BMPS_SIZECTLTOBMP

The size of the control is matched to the size of the bitmap that is displayed. Using this style leaves the dimensions of the bitmap unaltered and thus looks best.

BMPS_SIZEBMPTOCTL

The bitmap is shrunk or stretched to fit within the dimensions of the control.

BMPS_TRANSPARENT

The bitmap is displayed transparently on its background.

The following messages are available to bitmap display controls:

  • BMPM_SETFILE message
  • BMPM_SETHBITMAP message
  • BMPM_SETTRANSPARENTCOLOR message
  • BMPM_SETUSERHBITMAPMASK message

The following notification is available to bitmap display controls:

  • WM_COMMAND notification

Understanding counter controls

A counter control combines an edit control with increment and decrement buttons.

The control has two styles. The standard style makes it very easy to prompt the user for an integer within a given range. The owner-drawn style provides a visually consistent control that lets the user enter whatever data the application programmer desires.

The counter control, in its default style, allows the program to set a minimum value, a maximum value, a current value, and a step value. The scroll buttons do not increment or decrement the value beyond the valid range of values. If the control has the auto-validate style and the user enters a value outside the range, the value is automatically set to the nearest boundary value when the control loses focus.

The owner-drawn style of counter control provides a visually consistent control that lets the user enter anything in the edit area that can be entered in a standard edit control. The owner-drawn counter control notifies the owner when the user has clicked the increment or decrement buttons. It is the owner's responsibility to alter the contents of the counter control appropriately. The counter control handles WM_GETTEXT (that is, the GetWindowText( ) function) and WM_SETTEXT (that is, the SetWindowText( ) function) in the same way as a standard edit control; it is not possible to set a minimum, maximum, or step value for an owner-drawn counter control.

The class name for counter controls is WCWP_COUNTER.

The following styles are available to counter controls.

Style
Effect

CNTS_AUTOHSCROLL

The entered text is scrolled horizontally when the edge of the field is reached.

CNTS_AUTOVALIDATE

The value entered in the control is automatically validated when the control loses focus.

CNTS_NOSCROLL

The control is displayed without the increment/decrement buttons.

CNTS_OWNERDRAWN

The application is made responsible for handling the meaning of the increment/decrement buttons. The user is allowed to enter anything in the control.

Understanding date controls

If the button is clicked (or the F4 key is pressed), a dialog box appears from which the user can select a date. The selected date is displayed in the format chosen in the Windows control panel.

Understanding drag/drop well controls

A drag/drop well control looks like a recessed box with an icon in it. It handles the WordPerfect drag/drop protocol on behalf of the parent of the control. The user presses and holds the left mouse button to drag an object from this control to another window that can accept the dragged object. This control can also be a destination, so it can accept an object dragged and dropped onto it.

The class name for drag/drop well controls is WCWP_DDWELL. The window text is the name of the icon to be displayed in the control.

The following structure is available to drag/drop well controls:

  • DDWDATA structure

The following notification is available to drag/drop well controls:

  • WM_COMMAND notification
Understanding filename entry controls

A filename entry control appears as an edit field with a button to the right that has a file folder icon. If the button is clicked (or the F4 key is pressed), a dialog box appears allowing the user to select a file or a folder.

A filename entry control is used like a normal edit control. It responds to most of the same messages as an edit control and sends notifications to its parent that include those for an edit control. When you specify dimensions for the control, the button is placed inside the dimensions given, and the edit field portion of the control fills the remainder of the area.

The class name for filename entry controls is WCWP_FILENAME.

Understanding name search list box controls

A name search list box provides a way to search for an item in a list box by using the entire text of the item. This control is part of the dialog box manager system. A name search list box is a custom control that implements WordPerfect-style name search as a replacement for the Windows-standard first-character search. It can be used just like a standard list box in dialog boxes or windows. As each character is entered, a search string is displayed in a small window located above the list box - this window can be moved to a different location by using WPMSG_SET_NSLB_POPUP - and the best match is selected in the list box.

In a sorted list box, a binary search is performed through the list box to find the item that most closely matches the search string. If the list box is owner-drawn, the parent of the list box is asked to make the comparison through the WM_COMPAREITEM message. If the list box is not owner-drawn, then a case-insensitive lexical comparison is made. This comparison does not work if any of the characters that fall between `Z' and `a' - that is, the left brace ( [ ), the backslash ( \ ), the right brace ( ] ), the caret (^ ), the underscore ( _ ), or the apostrophe ( ' ) - appears in the list box item or the search string. A search is performed when a character is added or removed in the search pop-up.

In an unsorted list box, a sequential search is initiated from the item that currently has the focus or, if no item is selected, from the first item in the list box. The search wraps around from the end of the list box to the start of the list box if necessary. A match occurs when a perfect prefix match is found. If a perfect prefix match is not found, then the focus does not change. If the list box is owner-drawn, then the owner of the list box is asked to make the comparison though the WM_COMPAREITEM message. If the list box is not owner-drawn, then a case-insensitive lexical comparison is made. This comparison does not work if any of the characters that fall between `Z' and `a' - that is, the left brace ( [ ), the backslash ( \ ), the right brace ( ] ), the caret (^ ), the underscore ( _ ), or the apostrophe ( ' ) - appears in the list box item or the search string. A search is performed only when a character is added, not when one is removed.

In an owner-drawn list box, the owner of the list box is asked to perform each comparison through the WM_COMPAREITEM message. There is an override capability to terminate the search early; see the WM_COMPAREITEM message for details.

All previous selections are deselected with each character entered for a name search.

The name search pop-up appears on the first column of a multi-column list box, unless the position is changed though the WPMSG_SET_NSLB_POPUP message. When a selection is made, the list box ensures that the item selected is completely visible.

The class name for name search list box controls is WCWP_NS_LISTBOX.

The following messages are available to name search list box controls:

  • NSLB_CANCELPOPUP message
  • NSLB_QUERYPOPUP message
  • WM_COMPAREITEM message
  • WPMSG_SET_NSLB_POPUP message
Understanding pop-up/pop-down controls

A pop-up menu button control is a button that, when clicked, displays a menu from which selections can be made. The text on the button face is normally the currently checked menu item. On the right side of the button are small up/down arrows which, when pressed, typically display a menu over the button (with the selected item directly over the button).

A pop-down menu button control is a button that, when clicked, displays a menu from which selections can be made. The text on the button face can be set. On the right side of the button is a small down arrow which, when pressed, typically pops up a menu directly below the button.

A pop-down combo box control is an edit box with a button to its right. The button has a small arrow pointing at the edit box that, when clicked, displays a menu used to affect the contents of the edit box.

Working examples of the definition and use of the pop-up menu button and pop-down combo box controls are in the sample program.

The class names for these pop-up/pop-down controls are as follows:

  • pop-up menu button controls: WCWP_POPUP_BUTTON
  • pop-down menu button controls: WCWP_POPDOWN_BUTTON
  • pop-down combo box controls: WCWP_POPDOWN_COMBO

If a pop-up/pop-down control is used in a Dialog Box Manager (DBM) dialog box, the font of the dialog box is used to draw the control's text (rather than any font set via a WM_SETFONT message to the control) - in other words, the control uses the font obtained via a WM_GETFONT message sent to its parent. If a pop-up/pop-down control is not on a DBM dialog box, a WM_SETFONT can be sent to it to change the default font that is used for drawing text on the face of the button.

Understanding static path controls

A static path control is similar to a Windows static control. It can be used to display a path.

A static path control automatically shortens the path if the entire path does not fit within the space of the window. If a path must be displayed in a window as static text, using this control ensures that the filename at the end - the most important part of the path - is displayed. Any undisplayed part of the path is shown as an ellipsis ( ... ).

When given a path via WM_SETTEXT or SetWindowText( ), this control does not destroy the path that is sent to it, even though it may display only a portion of that path. If you get the text of the control, the full path (not a partial path with an ellipsis in the middle) is returned.

The class name for static path controls is WCWP_STATICPATH.

The following function is available to static path controls:

  • SquashFilename function

A static path control responds to the same messages as a Windows static control.

Understanding status bar controls

Status bar controls are part of the Dialog Box Manager (DBM) system. A status bar control provides a status bar at the bottom of a dialog box for displaying messages.

There are API functions available so that you can set up your own paint procedure to display in the status bar.

A status bar can be created in a dialog box via a macro provided in the SCWIN32.H file (which is stored at X:\Program Files\Corel\WordPerfect Office X6\SDK\Include, where X: is the drive where your Program Files folder is stored).

The class name for status bar controls is WCWP_SBAR.

Unlike for most controls, the ID for status bar controls is predefined. To update the text in the status bar, call the Windows API function SetWindowText( ) by using this predefined ID. This status bar ID is defined in the SCWIN32.H file as IDSBAR.

The following functions are available to status bar controls:

Understanding window editor controls

A window editor control is similar to a Windows edit control, except that it allows the entry of WordPerfect extended characters and function codes.

The class name for window editor controls is WCWP_EDIT.

On creation, the text field can contain "n,m" where n is the maximum number of characters that can be entered in the field and m (if present) is the buffer size (in bytes) to be used by the window editor. If m is not specified, the maximum buffer size of 2048 bytes is assumed. In almost all cases, m should not be specified because it does not reliably limit the amount of data that can be entered; the window editor works with WordPerfect document strings internally, and the buffer size relates to document strings. The only situation where you would specify m is when you have several window editor controls on a dialog box and you do not want to use 2048 bytes of memory for each one.

WES_NOTINDLG

WES_SOFTRET

Soft returns are left in strings.

To display functions as codes such as [BOLD] or [CENTER] in a window editor, you must use the WEM_FUNCTBL_6D message, where wParam contains the number of entries of WECODETAB and lParam points to array of WECODETAB.

All codes must be in the function range 0x80 0xFF and refer to WordPerfect document codes. (For word string codes, use the WEM_FUNCTBL_6W message.)

Do not use 0x99 0xA8. These are used internally.

Make sure that the functions in the table are in numerical order, and put the main function in the high byte and the subfunction (if any) in the low byte.

In the following example, all strings are displayed with ANSI characters only:

static WECODETAB fnctbl[] = { 
0x8000, "TEST", 
0xf000, "BOLD", 
0xd202, "DEF COL" }; 
SendMessage( hwnd, WEM_FUNCTBL_6D, 3, (LONG) fnctbl );

To insert individual codes in a window editor, use the function WEM_REPLACESEL_6D or WEM_REPLACESEL_6W.

In the following example, a bold code is inserted into the window editor at the current cursor position.

SendMessage( hEdit, WEM_REPLACESEL_6D, NULL, 
 (LONG)(LPSTR)"\xF0" );

Understanding WP list box controls

A WP list box control provides a list box that can accept and display WordPerfect word strings. It also provides the name-search capabilities described in "Understanding name search list box controls."

A WP list box is a custom control that operates in a way similar to the standard Windows list box control. However, instead of using ANSI strings, the WP list box uses WordPerfect word strings.

The class name for WP list box controls is WCWP_WPLISTBOX.

When WLBS_CHECKBOXES and WLBS_MULTIPLESEL are used, the name search does not select or deselect anything; rather, it just moves the focus rectangle to the item that matches the name search.
