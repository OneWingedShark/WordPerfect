Using PerfectFit : Providing access to viewer functionality through the View Manager : ViewMgrCreate function

ViewMgrCreate function

The ViewMgrCreate function creates a View Manager and Viewer window, using the specified location and size parameters. The Viewer window is created with a caption of "Viewer" and a blank client area. A handle to the View Manager is returned to the application. The application uses this handle in subsequent requests to the View Manager.

Prototype
HWND WvwrViewMgrCreate ( HWND hwndParent,   
WSWORD wnd_left,   
WSWORD wnd_top,   
WSWORD wnd_wd,   
WSWORD wnd_ht,   
BOOL visible,   
DWORD style );
Parameters

hwndParent

Specifies the handle to the parent window of the View Manager

wnd_left

Specifies the x-coordinate (in pixels) for the upper-left corner of the Viewer window

wnd_top

Specifies the y-coordinate (in pixels) for the upper-left corner of the Viewer window

wnd_wd

Specifies the width of the Viewer window (in pixels)

wnd_ht

Specifies the height of the Viewer window (in pixels)

visible

Indicates whether the Viewer window should be visible at this time. Usually, this value is TRUE. If it is FALSE, the application must call ShowWindow( ) to display the Viewer window.

style

Is used to specify any special window style bits necessary for this window. Usually this value is zero (NULL). If zero, the View Manager uses its default style for the window, which is as follows: WS_POPUP | WS_CAPTION | WS_SYSMENU | WS_THICKFRAME | _MAXIMIZEBOX. If a non-zero style is specified, all desired style bits should be specified. The standard style options for the Windows CreateWindow( ) API call should be used.

Return values

If successful, a non-zero handle to the View Manager is returned. This handle is needed in subsequent calls to the View Manager. If an error occurs, zero is returned.
