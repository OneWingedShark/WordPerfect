TPHMultipleInstance function

PerfectFit calls this function during DLL initialization. If this API is not implemented, WP will load your DLL only in the first running instance of WP.

This function does not have an exported ordinal value.

C Prototype
DWORD FAR PASCAL TPHMultipleInstance ( LPCSTR pazAppClassName );
Pascal Prototype
function TPHMultipleInstance ( const pazAppClassName: string ): DWORD;
Parameters

pazAppClassName 

Specifies the unique classname of the calling instance of WP.

The App classname should be saved to a static string, so that it can be used at a later time. A typical use for it is as a paramter for a FindWindow call. Without it, the wrong window handle will be returned.

Return values

The return value for the TPHMultipleInstance function should be a DWORD value of TRUE.

C sample
DWORD FAR PASCAL TPHMultipleInstance ( LPCSTR pazAppClassName ) 
{ 
 strcpy( g_szWPClassName, pazAppClassName ); 
 return 1L; 
}
Pascal sample
function TPHMultipleInstance ( const pazAppClassName: string ): DWORD;  
begin 
 psAppClassName := StrAlloc(128); 
 StrCopy (@psAppClassName, @pazAppClassName ); 
 Result := LongInt(1); 
end;
