DBMInsertPopupMenuItem function

The DBMInsertPopupMenuItem function inserts an item in the menu associated with the specified pop-up control.

Prototype
BOOL FAR PASCAL DBMInsertPopupMenuItem ( 
HWND hPopup, 
int nPosition, 
WORD usFlags, 
WORD usID, 
LPSTR lpID );
Parameters

hPopup

Specifies the handle to the pop-up control

nPosition

Specifies the position to insert the new menu item

usFlags

Takes the same flags as the Windows InsertMenu( ) function

usID

Specifies the identifier of the new menu item depending on the value of usFlags

lpID

Specifies the contents of the new menu item depending on the value of usFlags

The usFlags parameter takes the following flags.

MF_BITMAP

Uses a bitmap as the item, where the low word of lpID is the handle of the bitmap

MF_BYCOMMAND

Inserts the menu item at the position of a previously placed menu item's ID. The original menu item is moved down. This is the default.

MF_BYPOSITION

Inserts the menu item at a given position, where 0 is the first menu item

MF_DISABLED

Makes the menu item unselectable, but does not gray it

MF_ENABLED

Makes the menu item selectable and ungrays it if it has been grayed

MF_GRAYED

Makes the menu item unselectable and grays the text

MF_OWNERDRAW

Cause the WM_MEASUREITEM message to be sent to the window procedure that owns the menu the first time that the menu is displayed. This message requires the height and width of the owner-drawn menu item. Each time the menu is redisplayed, a WM_DRAWITEM message is passed to this same procedure. This message allows the program to update the owner-drawn item as necessary.

MF_SEPARATOR

Adds a horizontal line between menu items. Cannot be enabled, grayed, or selected. When this flag is used, usID and lpID are ignored.

MF_STRING

Indicates the menu item is a string. If this flag is set, then lpID must be pointing to a string.

In each of the following groups, the flags listed are mutually exclusive and cannot be used together:

  • MF_BYCOMMAND and MF_BYPOSITION
  • MF_DISABLED, MF_ENABLED, and MF_GRAYED
  • MF_BITMAP, MF_STRING, MF_OWNERDRAW, and MF_SEPARATOR
Return values

This function returns TRUE if successful, FALSE otherwise.
