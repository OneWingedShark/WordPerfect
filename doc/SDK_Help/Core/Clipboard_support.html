Using PerfectFit : Providing Clipboard support

Providing Clipboard support

Clipboard support for WPGroup products is provided through a set of specifications for Clipboard formats and a set of functions that work with the standard Windows Clipboard APIs.

Applications using PerfectFit require support for a wide variety of Clipboard formats (graphics, OLE and DDE objects, and sound capability, as well as text formats). Applications must also have control over the order in which formats are placed on the Clipboard.

Understanding Clipboard file formats

In addition to all of the standard Clipboard file formats that are defined by the Windows operating system (CF_TEXT, CF_BITMAP, CF_METAFILEPICT, and so forth), the following Clipboard formats have been defined specific to Corel WordPerfect Office products.

CF_WP11DOC

"WordPerfect Document (11)"

This is the current Clipboard format for WordPerfect 11 documents. This format is a DWORD-length field indicating the length of the WordPerfect document, followed by a complete document in WordPerfect format. The document data includes a document prefix, including any prefix packets required, followed by the body of the document. For a complete description of the WordPerfect 11 file format, see the File Format Software Developerment Kit.

CF_WP6DOC

"WordPerfect Document (6.x)"

This Clipboard format was used by WordPerfect 6.x for Windows and applications using shared code 1.x. WPGroup will not use it in any future products and recommends that developers not use this format. This format is a DWORD-length field indicating the length of the WordPerfect document, followed by a complete document in WordPerfect format. For a complete description of the WordPerfect 6.x document format, see the File Format Software Development Kit.

CF_WPDOC

"WordPerfect Document"

This Clipboard format was used by WordPerfect 5.x for Windows and applications using shared code 1.x. WPGroup will not use it in any future products and recommends that developers not use this format. This format is essentially a complete WordPerfect 5.x document as it would be stored on disk, except that it is stored in a global memory block. The memory block must be null filled at the end. This format is the same as CF_WP6DOC except that no length is indicated at the beginning.

CF_WP11TEXT

"WordPerfect Text (11)"

This Clipboard format is a WordPerfect 11 word-string terminated with two null bytes. Two-byte function codes may also be included. See the STRFUNCS.HTM file included with the PerfectFit Software Development Kit for a complete list of two-byte function codes and their values.

CF_WPTEXT

"WordPerfect Text"

This Clipboard format contains WordPerfect 5.x text only (no prefix, limited function codes, null-terminated). It is used by WordPerfect 5.x for Windows and possibly by other products that use PerfectFit. It is recommended that developers not use this format.

CF_RTF

"Rich Text Format"

This Clipboard format is the Microsoft Rich Text Format (.rtf).

CF_WPG20

 

This is the Clipboard format for WordPerfect graphics (.wpg). Its format is the same as a .wpg 2.0 file.

Understanding the functions for providing Clipboard support

The following functions are contained in the PerfectFit image:

  • WclpIsWPTextFormatAvailable function
  • WclpIsWPGraphicFormatAvailable function
  • WclpCFtoCV function
  • WcvtConvertHandle function

To use these functions, an application must first register with PerfectFit. For information on registering PerfectFit, see "Registering PerfectFit."
